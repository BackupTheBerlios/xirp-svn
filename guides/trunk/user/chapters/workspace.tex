\chapter{Arbeitsbereich}
\index{Arbeitsbereich|(}
Der Arbeitsbereich lässt sich in fünf Bereiche aufteilen. Im folgenden werden
diese fünf Bereich getrennt voneinander erläutert.

\begin{itemize}
  \item Menü
  \item Anwendungswerkzeugleiste
  \item Roboterwerkzeugleiste
  \item Statusleiste
  \item Drag-and-Drop Hauptarbeitsbereich
\end{itemize}

Im folgenden Kapitel werden diese Punkte einzeln beschrieben und näher erläutert.

\subsection{Datei}
\index{Arbeitsbereich!Menü!Datei}
\index{Kommunikationsspezifikationsdatei}
\index{Roboterdatei}

Das Menü \enquote{Datei} enthält acht Einträge. Allerdings sind von diesen in
der Version 2.4.0 nur vier verfügbar, da momentan noch keine Editoren für
Roboter- und Kommunikationsspezifikationsdateien existieren
\footnote{Es ist geplant, dass diese Editoren ab der Version 3.0.0 verfügbar
seien werden.}. Daher fallen die Einträge \texttt{Neuer Roboter},
\texttt{Roboter bearbeiten}, \texttt{Neue Kommunikationsspezifikation} und
\texttt{Kommunikationsspezifikation bearbeiten} weg. Die restlichen vier
Einträge werden im folgenden beschrieben.

\subsubsection{Neues Profil}
\index{Profile!Profileditor}
Wird dieser Eintrag ausgewählt, öffnet sich der Profileditor im Modus zum
Erstellen eines neuen Profils.

\subsubsection{Profil bearbeiten}
\index{Profile!Profileditor}
Wird dieser Eintrag ausgewählt, öffnet sich der Profileditor im Modus zum
Bearbeiten eines bereits kompletten, oder eines nicht kompletten Profils.

\subsubsection{Einstellungen}
\index{Einstellungen}
Die Auswahl dieses Eintrages öffnet den Einstellungsdialog. Mehr über die
vorhandenen Einstellungsmöglichkeiten kann im Kapitel \enquote{Einstellungen}
ab Seite \pageref{ref:settings} nachgelesen werden.

\subsubsection{Beenden}
Wird dieser Eintrag gewählt schließt sich die Anwendung.

\subsection{Ansicht}
\index{Arbeitsbereich!Menü!Ansicht}

Das Menü \enquote{Ansicht} enthält sieben Optionen die aktiviert oder
deaktiviert werden können. Jeder dieser Optionen bewirkt ein sichtbare Änderung
im Arbeitsbereich.

\subsubsection{C.O.P. sichtbar}
\index{Kontrollübersicht}
Diese Option ist nur verfügbar, wenn mindestens ein Profil mit mehr als einem
Roboter geladen wurde. Wird die Option aktiviert ist die Kontrollübersicht als
weiterer Reiter auf der Profil-Reiter Ebene verfügbar. Wird die Option
deaktiviert wird auch die Kontrollübersicht geschlossen.

\subsubsection{Werkzeuge sichtbar}
Wird diese Option aktiviert ist die Anwendungswerkzeugleiste sichtbar, wird sie
deaktiviert wird diese unsichtbar und stellt dem Arbeitsbereich etwas mehr
Platz zur Verfügung.

\subsubsection{Werkzeuge arretiert}
Wird diese Option aktiviert wird die Anwendungswerkzeugleiste arretiert, wird
sie deaktiviert ist es möglich die Werkzeuge in der Werkzeugleiste zu
verschieben.

\subsubsection{Statusleiste sichtbar}
Wird diese Option aktiviert ist die Statusleiste sichtbar,wird sie
deaktiviert wird diese unsichtbar und stellt dem Arbeitsbereich etwas  mehr
Platz zur Verfügung.

\subsubsection{Systemprotokolle sichtbar}
\index{Systemprotokoll}
Wird diese Option aktiviert sind in allen vorhandenen Roboter-Reitern die
Systemprotokolle\footnote{Ab Version 3.0.0 wird diese Option profilabhängig. So
werden nicht alle Systemprotokolle aus allen Reitern entfernt, sondern nur für
ein bestimmtes Profil.} sichtbar, wird sie deaktiviert werden die Protokolle
nicht angezeigt.

\subsubsection{Live-Diagramme sichtbar}
\index{Live-Diagramme}
Wird diese Option aktiviert sind in allen vorhandenen Roboter-Reitern die
Live-Diagramme\footnote{Ab Version 3.0.0 wird diese Option profilabhängig. So
werden nicht alle Live-Diagramme aus allen Reitern entfernt, sondern nur für
ein bestimmtes Profil.} sichtbar, wird sie deaktiviert werden die
Live-Diagramme nicht angezeigt.

\subsubsection{Rekorder sichtbar}
\index{Rekorder}
Wird diese Option aktiviert sind in allen vorhandenen Roboter-Reitern die
Rekorder\footnote{Ab Version 3.0.0 wird diese Option profilabhängig. So
werden nicht alle Rekorder aus allen Reitern entfernt, sondern nur für
ein bestimmtes Profil.} sichtbar, wird sie deaktiviert werden die Rekorder nicht
angezeigt.

\subsubsection{Roboterübersicht sichtbar}
\index{Roboterübersicht}
Wird diese Option aktiviert sind in allen vorhandenen Roboter-Reitern die
Roboterübersicht\footnote{Ab Version 3.0.0 wird diese Option profilabhängig. So
werden nicht alle Roboterübersichten aus allen Reitern entfernt, sondern nur für
ein bestimmtes Profil.} sichtbar, wird sie deaktiviert werden die
Roboterübersichten nicht angezeigt.

\subsection{Roboter}
\index{Arbeitsbereich!Menü!Roboter}
Das Menü \enquote{Roboter} ist ein dynamisch beim Start der Anwendung erstelltes
Menü. Es enthält für jedes erfolgreich geladene Profil einen eigenen Eintrag.
Öffnet man eines dieser Menüs wird für jeden Roboter des Profils eine Option
angezeigt\footnote{Ab Version 3.0.0 wird es die Möglichkeit geben den Zustand
des Profils bei Beenden der Anwendung im Profil zu speichern.}. Ist diese Option
aktiviert wird der entsprechende Roboter-Reiter angezeigt, wird sie deaktiviert wird
der Roboter-Reiter geschlossen.

\subsection{Plugins}
\index{Arbeitsbereich!Menü!Plugins}
\index{Plugins}
\index{Einstellungen}
Das Menü \enquote{Plugins} ist ein dynamisches, kontextsensitives Menü. Es ist
abhängig von dem aktuell aktivem Roboter. Es enthält Einträge für die Plugins
des Roboters, welche entweder keine graphische Benutzeroberfläche haben und nur
eine Funktionalität ausführen, oder solche die zwar eine graphische
Benutzeroberfläche besitzen, aber nicht eingebettet in den Arbeitsbereich
dargestellt werden sollen, sondern in Dialogform in einem eigenen Fenster.
Des Weiteren enthält dieses einen Eintrag der die Einstellungen öffnet.

\subsection{Reporte}
\index{Arbeitsbereich!Menü!Reporte}
\index{Reporte}
\index{Reporte!PDF}
\index{Plugins}
Das Menü \enquote{Reporte} ist ein dynamisches, kontextsensitives Menü. Es ist
abhängig von dem aktuell aktivem Roboter. Es enthält Einträge für die
Report-Plugins des Roboters, sowie einen Eintrag zum Öffnen des Suchdialogs für
Reporte. Wird ein Report erzeugt, wird dieser nach der Erstellung sofort im
\seegls{PDF}-Betrachter des Betriebssystems geöffnet.

\subsection{Extras}
\index{Arbeitsbereich!Menü!Extras}
Das Menü \enquote{Extras} enthält Einträge für diverse zusätzliche Funktionen
die in \xirp~verfügbar sind.

\subsubsection{Mail}
\index{Mail}
Wird dieser Eintrag gewählt öffnet sich die Mailverwaltung der Anwendung.

\subsubsection{Kontakte}
\index{Kontakte}
Wird dieser Eintrag gewählt öffnet sich die Kontaktverwaltung der Anwendung.

\subsubsection{Diagramme}
\index{Diagramme}
Wird dieser Eintrag gewählt öffnet sich der Dialog zum Konfigurieren des
Diagrammgenerators.

\subsubsection{Testerbot ausführen}
\index{Testerbot}
Wird dieser Eintrag gewählt öffnet sich der Kontrolldialog des \seegls{Testerbot}
\seegls{Server}s.

\subsection{Externe Werkzeuge}
\index{Arbeitsbereich!Menü!Externe Werkzeuge}
\index{Externe Werkzeuge}
Das Menü \enquote{Externe Werkzeuge} ist ein dynamisches, kontextsensitives
Menü. Es ist abhängig vom aktuell aktivem Profil. Es enthält für jedes im Profil
spezifiziertes externes Werkzeug einen Eintrag mit dem angegebenen Namen des
Werkzeugs. Wird ein Eintrag ausgewählt werden die in dem externen Werkzeug
angegebenen Programme in der gewollten Reihenfolge und in dem angegebenen
zeitlichen Abstand gestartet.

\subsection{?}
\index{Arbeitsbereich!Menü!?}
Das Menü \enquote{?} enthält Einträge zum Anzeigen von Programminformationen und
der Hilfe zur Anwendung und den Plugins.

\subsubsection{Über \xirp~2}
Wird dieser Eintrag gewählt öffnet sich ein Informationsdialog, der die aktuelle
Versionnummer anzeigt.

\subsubsection{Über Plugins}
\index{Plugins}
Wird dieser Eintrag gewählt öffnet sich ein Dialog, der Informationen über die
geladenen Plugins anzeigt.

\subsubsection{Hilfe}
\index{Hilfe}
Wird dieser Eintrag gewählt öffnet sich die Hilfe zu \xirp~und den geladenen Plugins.

\subsection{Drag-and-Drop der enthaltenen Elemente}
\index{Plugins}
Der Arbeitsbereich eines Roboters setzt sich aus den Standard Elementen und den
Plugin Elementen zusammen. Die Elemente können frei in diesem Bereich angeordnet
werden. Mittels \seegls{Drag-and-Drop} kann die Anordnung der Elemente verändert werden.
Um z.B. Element \enquote{A} links neben Element \enquote{B} zu verschieben muss
die Maus über der Titelzeile des Element gehalten und dann auf die
Trennlinie links neben Element \enquote{B} verschoben und dann
\enquote{losgelassen} werden. Pfeile zeigen an, ob ein Element an der
gewünschten Stelle eingefügt werden kann. Es ist ebenfalls möglich Elemente
übereinander zu \enquote{stapeln} in dem das Element einfach über einem andern
\enquote{losgelassen} wird.

\subsection{Anzeigemöglichkeiten der Elemente}

Fenster angezeigt werden. Um dieses zu erreichen muss auf das \enquote{in

\ref{img:windowed} auf Seite \pageref{img:windowed} rot markiert. Die Abbildung
zeigt ebenfalls, wie ein Plugin in einem eigenen Fenster angezeigt wird. Das
Plugin wird, wenn das Fenster geschlossen wird, wieder an seine ursprüngliche
