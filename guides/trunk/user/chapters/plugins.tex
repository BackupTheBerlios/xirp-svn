\chapter{Standard-Plugins}
\index{Plugins|(}
\index{Testerbot}
Die Installation von \xirp~enthält die nötigen Profildateien für den
\seegls{Testerbot}. Ebenfalls werden einige Plugins mitgeliefert, die für den \seegls{Testerbot}
benötigt werden. In diesem Kapitel werden diese Plugins vorgestellt und näher
beschrieben. Des Weiteren werden einige Kommunikationsplugins mitgeliefert, z.B.
ein Plugin für Kommunikation über die serielle Schnittstelle.

\newpage

\section{Hallo Welt!}
Das \enquote{Hallo Welt!} Plugin ist das erste Plugin, dass je für
\xirp~entwickelt wurde. In seiner momentanen Version ist dieses Plugin
ohne graphische Oberfläche. Es wird mittels seines Menüeintrages \texttt{Plugins
--> Hallo Welt!} gestartet und gibt in der Systemprotokollansicht einen in den
Einstellungen änderbaren Text an. Der Standardtext ist \enquote{Hallo Welt!}.
Diese Funktion kann ebenfalls mit dem vom Plugin erzeugten Werkzeugicon
aufgerufen werden. Das \seegls{Icon} ist gekennzeichnet durch ein Puzzelteilbild.

\section{Testerbot Report}
Dieses Plugin zeigt die Funktionsweise von Report-Plugins. Es erzeugt, bei Aufruf
über den entsprechenden Menüeintrag (\texttt{Reporte --> Testerbot Report}),
einen Report der die übertragenen Textnachrichten darstellt.

\index{Abstandssensor!Infrarot}
\index{Abstandssensor!Ultraschall}
\index{Roboterprofil}
Infratorsensoren werden farblich anders dargestellt als Ultraschallsensoren.
Infratorsensoren werden grün und Ultraschallsensoren blau dargestellt, wenn der
Entfernungswert größer oder gleich dem Maximum des \seegls{Sensor}s entspricht. Die Farbe
der Sensoranzeigen variiert mit den sich verändernden Werten. Die Einheit in der
die Werte angezeigt werden, werden ebenfalls aus dem Roboterprofil ausgelesen.

\section{Nachrichten}
\index{Testerbot}
\index{Sprachausgabe}
\index{Kommandos}
Das Nachrichten-Plugin zeigt die vom \seegls{Testerbot} erzeugten Sätze an. Ist die
Sprachausgabe aktiviert, werden diese Sätze (alle in englischer Sprache) über
das Sprachausgabesystem ausgegeben. Des Weiteren exportiert dieses Plugin eine
Funktion, welche mittels der Kommandos aufgerufen werden kann. Wird dieser
Funktion eine Taste zugewiesen, kann durch Drücken dieser Taste der Inhalt des
Textfensters gelöscht werden.

Das \seegls{Testerbot} Protokoll verarbeitet die vom \seegls{Server} gesendeten Pakete und spielt

\section{IP Kommunikation}
\index{TCP/IP}
\index{Testerbot}
Dieses Kommunikationsmediumsplugin kann verwendet werden, wenn mit dem Roboter
via \seegls{TCP/IP} kommuniziert werden soll. Der \seegls{Testerbot} kann ausschließlich mit
diesem Medium angesprochen werden.
\par
\index{IP-Adresse}
In den Einstellungen ist es möglich die \seegls{IP-Adresse} und den Port des
Roboters/\seegls{Server}s einzugeben. Die \seegls{IP-Adresse}n werden in IPv4 angegeben.

\section{Serielle Kommunikation}
\index{RS232}
Dieses Plugin realisiert eine Kommunikation über eine serielle Schnittstelle
(RS232). In den Einstellungen kann die zu benutzende Schnittstelle definiert und
konfiguriert werden.

\section{Carmen Kommunikation}
\index{Carmen}
\index{Carmen!IPC}
\index{Carmen!Carmen IPC Central Server}
Mit diesem Plugin lässt sich eine Verbindung zu einem \seegls{Carmen} \seegls{IPC} Central \seegls{Server}
herstellen und Nachrichten an diesen verschicken. In den Einstellungen lassen
sich \seegls{IP-Adresse} und Port des \seegls{Server}s angeben.
