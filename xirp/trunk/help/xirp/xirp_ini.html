χirp Configuration

The main configuration if χirp is done in the xirp.ini file located in the conf folder of the installation folder of χirp.
There are 9 sections with options available in the xirp.ini file. Each of those sections are described below. Every value in the xirp.ini can be set using the preferences dialog.

General
The [General] section contains options about the general behavior of the application. At the moment there are 3 options available:

  • locale
    This option indicates the language that χirp should use. If the language pack does not exist, the default language (english) is used.

  • timerWarning
    This options sets a time value in milliseconds. This value indicates when the robot timers should change to their warning mode.

  • timerFinalThis options sets a time value in milliseconds. This value indicates the maximum time of the robot timer.


If you want to know more about the robot timers, please read the Workspace chapter.

Mail
The [Mail] section contains options for configuring the integrated mailing system.

  • smtpHost
    This option holds the host name of the SMTP server to use.

  • smtpUser
    This options holds the user name for the SMTP server to use.

  • smtpPassword
    This options holds the encrypted password to use. This options can not ne set directly in the xirp.ini file. It must be set via the prefences dialog.

  • needsAuthentication
    A flag indicating if the SMTP server needs an authentication.

  • smtpPort
    This options holds the port number on which the SMTP server is reachable.

  • noReplyAddress
    This option holds the noreply mail address.



Appearance
The [Appearance] section contains color options. Seven options are used to configure the appearance of the application. The best way to set those values is to use the preferences dialog.

  • tabColor
    The color of a tab in the workspace.

  • tabTextColor
    The color of the text on a tab in the workspace.

  • inactiveTabTextColor
    The color of the text on an inactive tab in the workspace.

  • focusColor
    The color which is set to controls gaining the focus, f.e. text fields.

  • sashColor
    The color of the sashes in the workspace.

  • backgroundColor
    The background color for all controls of the application.

  • panelHeaderColor
    The color for the panel headers in the workspace.

Database
The [Database] section contains informations about the connection to a database and the used database driver.

  • dbIP
    The IP address of the database server, if one is used.

  • dbIP
    The port number where the server is reachable.

  • dbIP
    The username to use.

  • dbIP
    The encypted password to use. This option must be the via the preferences dialog.

  • dbIP
    The database driver to use. This option must be set via the preferences dialog.


The default database is the HSQLDB running in file mode. Only the driver must be selected to have this database up and running. Read more about the database in the database chapter.

Hotkeys
The [Hotkeys - *] section contains the shortcuts for several functions.

  • program_quit
    Quit the application using CTRL+[KEY]

  • program_preferences
    Open the preferences using CTRL+[KEY]

  • program_help
    Open the help using CTRL+[KEY]

  • open_mail
    Open the mail dialog using CTRL+[KEY]

  • create_chart
    Open the charting dialog using CTRL+[KEY]

  • plugin_info
    Show the info about the loaded plugins using CTRL+ALT+[KEY]

  • manage_contacts
    Open the contact managing dialog using CTRL+ALT+[KEY]

  • search_reports
    Open the report search using CTRL+ALT+[KEY]

  • program_info
    program_infoShow the info about the application using CTRL+SHIFT+[KEY]



Speech
The [Speech] section contains options for configuring the speech system.

  • enable_tts
    A flag indicating if the text to speech system should be active or not.

  • tts_voice
    The name of the language to use. This option should be set via the preferences dialog.



Chart
The [Chart] section contains options to configure the export system of the live charting. Those flags indicate whether or not a file should be exported automatically.

  • export_pdf
    A flag indicating if a PDF should be generated automatically after a live charting has been stopped.

  • export_png
    A flag indicating if a PNG should be generated automatically after a live charting has been stopped.

  • export_jpg
    A flag indicating if a JPG should be generated automatically after a live charting has been stopped.

  • export_csv
    A flag indicating if a CSV should be generated automatically after a live charting has been stopped.
